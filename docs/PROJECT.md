## 1. 产品理念

### 你的角色 vs 系统的角色

```
你（导演）
│
├── 定世界观 → 存入知识库（不可违背的规则）
├── 定人物 → 存入知识库（演员的性格和禁忌）
├── 定剧情走向 → 存入知识库（剧本大纲）
├── 定风格 → 存入风格指南（腔调、句式、用词）
├── 写章节节拍 → 交给写作流程（这一章的情节骨架）
└── 选本章意图 → 交给写作流程（这一章的情绪和重点）
                      │
                      ▼
系统（执行制片）
│
每当你要写一章时，系统自动：
├── 从知识库取出本章需要的内容（风格指南+故事圣经+角色+设定）
├── 从前几章日志中取出前情提要（前3章摘要+故事线进度）
├── 取出你写的节拍和意图
├── 组装成一份 ~4,200 tokens 的章节指令
├── 发给 AI → AI 写完 → 你审阅 → 定稿
└── 定稿后自动更新日志（生成摘要/更新角色状态/更新故事线/管理伏笔）
```

### 核心原则

| 原则 | 说明 |
|------|------|
| **逐章写作** | AI 一次只写一章，你审阅定稿后再写下一章 |
| **分层上下文** | 每次只发本章相关的数据，不塞全部。600章时和10章时发给AI的信息量一样 |
| **不打破铁则** | 风格指南+故事圣经每次必带，AI 不能违反 |
| **可追溯** | 每章摘要+原文快照保留，质量检查器可回溯验证 |
| **设定可变** | 知识库支持版本记录，变更后标记生效章节 |

## 3. 创作流程

### 全程三段式

```
阶段一：设定（知识库建设）
─────────────────────────────
你在各模块中完成：
  · 世界观画布 → 词条管理
  · 人物星图 → 角色卡 + 关系网
  · 剧情走向 → 明暗线 + 时间轴节点
  · 故事圣经 → 风格指南 + 铁则 + 里程碑

阶段二：逐章写作（核心循环）
─────────────────────────────
每写一章，走一遍完整循环：
  ① 你在分章节拍中写节拍卡片
  ② 你切换到写作台，选中本章
  ③ 系统自动组装上下文 → 发给 AI
  ④ AI 写出本章 → 你审阅
  ⑤ 定稿 → 系统自动更新日志
  ⑥ 下一章开始，重复

阶段三：审阅与修正
─────────────────────────────
每10章或你觉得需要时：
  · 质量检查器自动扫描已写章节
  · 检查角色性格是否漂移
  · 检查伏笔是否该收还没收
  · 检查是否违反故事圣经
```

```
│                                                        │
│  → 组装成 7 部分上下文                                  │
│  → 裁剪到 ~4,200 tokens                                 │
│  → 输出: 完整的 system_hint                              │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌────────── ② 发送给 AI ─────────────────────────────────┐
│                                                        │
│  POST DeepSeek API                                      │
│  messages: [                                            │
│    { role: "system", content: "①~⑦ ~4,200t" },         │

│                                                        │
│  → AI 返回本章正文                                      │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌────────── ③ 你审阅 ────────────────────────────────────┐
│                                                        │
│  AI 写好的内容展示在编辑器中                              │
│  ┌─── 满意 → 点击"定稿"                                 │
```

### 7.1 上下文引擎 (`lib/context-engine.ts`)

**职责**：组装每次 AI 请求的分层上下文

```typescript
// 核心接口
interface ContextEngineInput {
  projectId: string;
  chapterId: string;
  userIntent?: WritingIntent;  // 你选的本章意图（选填）
}

interface ContextEngineOutput {
  layers: {
    p0: string;  // 风格指南 + 故事圣经（不可裁剪）
    p1: string;  // 前情提要 + 节拍 + 意图（可压缩）
    p2: string;  // 活跃角色卡（可按字段裁剪）
    p3: string;  // 相关设定 + 全局状态（可省略）
  };
  totalTokens: number;
  omitted: string[];  // 因 token 限制跳过的内容清单
}

async function buildProjectContext(input: ContextEngineInput): Promise<ContextEngineOutput>;
```

**相关性过滤规则**（硬编码，不由 AI 决定）：

| 取什么 | 匹配方式 | 默认上限 |
|--------|---------|---------|
| 出场角色 | 节拍中提到的角色名 | 3-5 人 |
| 涉及世界观 | 节拍中匹配的词条标题 | 2-3 条 |
| 前几章摘要 | 固定取前 3 章 | 3 章 |
| 相关故事线 | 节拍中匹配的故事线名 | 1-2 条 |
| 相关伏笔 | 本章编号在伏笔的预期章节范围 | 1-2 条 |

**裁剪策略**：

```
总 token 预算上限: 5,000

P0 (不可裁): 风格指南(~500) + 故事圣经(~500) = ~1,000
P1 (压缩):   前3章摘要(~600→压到400) + 节拍(~200) + 意图(~200) = ~800
P2 (裁剪字段): 3人×~400→每人只保留关键字段: 性格/状态/禁忌 = ~600
P3 (可选):    相关设定(~300) + 全局状态(~200) = ~500

总量: ~2,900 tokens ← 安全范围内

当项目更大时，从 P3 开始砍，确保 P0+P1 始终完整
```

### 7.2 质量检查器 (`lib/quality-checker.ts`)

**职责**：定稿后自动检查，确保不偏移。自动运行，不可跳过。

| 检查项 | 对比源 | 严重程度 | 示例 |
|--------|--------|---------|------|
| 圣经合规 | 故事圣经.铁则 | error | 叶玄杀无辜 → 违反"不滥杀"铁则 |
| 角色一致 | 角色档案.性格/能力 | warning | 叶玄展现金丹期实力但他是筑基初期 |
| 伏笔回收 | 伏笔清单.预期章节 | warning | 某伏笔预期42-55章回收，现已43章未触发 |
| 版本正确 | 知识库.版本记录 | error | 使用已过期的 v1.0 设定 |

```
检查结果处理:
  ┌─── 全部 pass → 静默，不打扰你
  ├─── 有 warning → 静默记录，在审计报告中提示
```
