# 世界观编组位置诊断脚本

## 使用方法

### 方法1: 在 EXE 应用中运行（推荐）

1. 打开知彼应用
2. 进入一个项目
3. 按 `F12` 或 `Ctrl+Shift+I` 打开开发者工具
4. 切换到 "Console" (控制台) 标签
5. 复制 `diagnose-groups-advanced.js` 的内容
6. 粘贴到控制台并按 Enter 运行

### 方法2: 在浏览器预览中运行

1. 运行 `npm run dev` 启动开发服务器
2. 在浏览器中打开应用
3. 按 `F12` 打开开发者工具
4. 切换到 "Console" 标签
5. 复制 `diagnose-groups.js` 的内容
6. 粘贴到控制台并按 Enter 运行

## 诊断内容

脚本会检查以下内容：

1. **编组数据完整性** - 检查编组的 ID、名称、坐标是否有效
2. **子节点存在性** - 检查编组中的子节点是否在数据库中存在
3. **坐标一致性** - 检查编组内词条的绝对坐标和相对坐标
4. **重复编组检查** - 检查是否有词条同时属于多个编组

## 修复函数

高级诊断脚本提供以下修复函数：

```javascript
// 清理孤立的子节点引用
cleanOrphanChildren()

// 修复重复编组问题
fixDuplicateGroups()

// 重置所有编组数据（谨慎使用）
resetGroups()
```

## 常见问题

### Q: 编组位置重置怎么办？
A: 运行诊断脚本检查坐标一致性，如果发现问题，使用 `resetGroups()` 重置后重新创建编组。

### Q: 切换页面后编组位置混乱？
A: 这通常是由于 `onNodesChange` 没有正确保存编组位置。已在此版本中修复。

### Q: 对齐/等距后位置丢失？
A: 这是由于 `align`/`dist` 函数没有持久化。已在此版本中修复。
